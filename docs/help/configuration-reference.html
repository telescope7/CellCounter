Tracking Configuration Reference

Detailed behavior of every tunable tracking parameter, including value-direction effects.

Where Configuration Is Applied

  • GUI session: Tune Detection dialog modifies active runtime parameters.
  • Startup defaults: CellCounter.properties is loaded at launch for GUI and headless.
  • Headless override: CLI flags and --tracking-config=<file> can override startup defaults.
If CellCounter.properties is missing, built-in defaults from code are used.

Parameter-by-Parameter Effects

Key Description If Increased If Decreased
maxFramesDisappeared Frames a track can be missing before it is dropped. Tracks survive occlusion longer; less fragmentation. Tracks terminate faster; fewer stale IDs.
minContourArea Minimum contour area to be treated as a cell candidate. Rejects small noise, but can miss tiny/dim cells. Finds smaller objects, with higher false positives.
maxRectCircumference Maximum allowed contour bounding-rectangle perimeter. Allows larger blobs/merged shapes to pass. Suppresses larger artifacts/merged cells.
maxVerticalDisplacementPixels Allowed vertical movement per association step. Tolerates Y jitter/drift and tilted flow. Enforces straighter horizontal rolling.
minHorizontalMovementPixels Minimum horizontal movement for forward motion acceptance. Stricter movement requirement; may drop slow movers. More permissive; can admit stalls/backward jitter.
maxAssociationDistancePixels Max centroid distance to link detections into one track. Maintains tracks through larger jumps; more mismatch risk. Tighter ID matching; more track breaks when motion is large.
mog2HistoryFrames Background model history depth for MOG2. More stable background, slower adaptation. Faster adaptation, can absorb slow/standing cells.
mog2VarThreshold MOG2 variance threshold for foreground decision. More conservative segmentation (fewer detections). More sensitive segmentation (more detections/noise).
mog2DetectShadows Enable/disable MOG2 shadow detection mode. When true, labels possible shadows separately; can reduce some artifacts. When false, simpler foreground model; often preferred for high-contrast captures.
morphologyKernelSize Kernel size for morphological ops (auto-normalized to odd). Smoother blobs; can merge nearby objects. Preserves detail; can leave noisy fragments.
morphologyOpenIterations Opening passes to remove small bright speckles. More noise cleanup; can erode weak real objects. Less cleanup; more speckle survives.
morphologyDilateIterations Dilation passes to expand detected foreground regions. Bridges gaps and strengthens blobs; can over-merge. Less expansion; fragmented detections remain separate.
normalizedMaskThreshold Threshold applied to normalized mask (0 to 255). Stricter foreground acceptance; less noise, lower recall. More permissive foreground acceptance; higher recall/noise.

Normalization and Type Rules

  • maxFramesDisappeared, mog2HistoryFrames, and morphology iteration counts are integer values.
  • morphologyKernelSize is forced to odd and at least 1.
  • normalizedMaskThreshold is clamped to [0, 255].
  • Negative minHorizontalMovementPixels values are valid.
