Standard Workflow Analysis

From opening a video to tuning tracking and saving outputs.

Back to Help Home

1. Open the Source Video

Click Open Video, select a microscopy video, and verify the first frame renders in the Video panel. Status moves from Idle to Loaded.

Use the frame slider immediately after loading to move to a representative frame before tuning.

2. Configure/Tune Detection Parameters

Use Tune Detection while paused. Adjust sliders and release to render a live preview of segmentation/tracking on the current frame.

  • Enable Preview mask view to inspect foreground extraction quality.
  • Adjust sensitivity/noise controls first: MOG2 settings, threshold, contour area.
  • Then adjust association rules: distance and movement/disappearance constraints.
  • Click Apply Parameters to make values active for the session.

3. Analyze the Video

Choose one of two analysis paths:

  • Play/Analyze: live playback and analysis.
  • Fast Analyze: non-realtime batch pass for speed.

Use Step for frame-by-frame checks, and Replay to restart from frame 1 with current settings.

4. Review Metrics and Distribution Charts

The right-side charts update with track-start times and speed values as analysis progresses.

  • Track Start Distribution: when tracks begin over time.
  • Speed Distribution: observed instantaneous speeds.

5. Save Results

Click Save Results, provide metadata, and choose a base filename. The app writes both outputs:

  • <base>_analysis.csv
  • <base>_footprint.csv
Re-run analysis after major tuning changes to keep exported results consistent with final parameters.

6. Optional Validation Workflow (Ground Truth)

  • Generate simulation videos/CSVs via Simulator.
  • Run CLI evaluators against truth files.
  • Use GA tuner for automated parameter search when needed.
./run-ground-truth-eval.sh --video=... --truth-events=... \
  --tracking-config=CellCounter.properties --output-prefix=/tmp/eval01

See UI Button Reference and Configuration Reference for parameter-level details.
