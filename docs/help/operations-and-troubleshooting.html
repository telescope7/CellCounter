Output Files and Meaning

FilePurpose
<base>_analysis.csv Primary analysis summary and per-track measurements used for reporting/statistics.
<base>_footprint.csv Detailed frame/position footprint records used for deeper validation and downstream analytics.
simcell_*.csv (Simulator) Ground-truth events/trajectory files used by evaluator and GA tuner workflows.

Recommended Tuning Order (for Noisy Real Videos)

  1. Pause on a frame with challenging background texture.
  2. Enable mask preview in Tune Detection.
  3. Raise minContourArea until background speckles are mostly removed.
  4. Adjust normalizedMaskThreshold and mog2VarThreshold for clean object fill.
  5. Refine morphology settings to remove speckle without merging neighboring cells.
  6. Then tune association/motion parameters to improve ID continuity.

Ground-Truth Evaluation Workflow

./run-ground-truth-eval.sh \
  --video=/path/simcell.avi \
  --truth-events=/path/simcell_events.csv \
  --tracking-config=/path/CellCounter.properties \
  --output-prefix=/tmp/eval_run

The evaluator computes comparable quality metrics (including score/F1/time/velocity terms) against truth files.

Common Issues

SymptomLikely CauseAction
Help button shows “documentation not found” Help files not present in expected app/project location Ensure docs/help exists in source and packaged help/ is copied in distribution.
OpenCV load error Native library not discoverable Use packaged app output or validate OpenCV runtime dependency and loader fallback.
Too many tiny false detections Threshold/area too permissive Increase minContourArea, increase normalizedMaskThreshold, adjust morphology opening.
IDs break too often Association constraints too strict Increase maxAssociationDistancePixels and/or maxFramesDisappeared.
Different score than previous GA run Random seed, timeout, or frame subset differences Fix seed, keep same timeout/max-frames, and rerun with identical inputs.

Distribution and Packaging

Use the project packaging script to bundle app jars, runtime dependencies, properties, and help docs:

./package-app.sh --type=app-image

Packaged builds include local help HTML and can open it via the in-app Help button.
